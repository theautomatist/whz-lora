# Onboarding

Conduct onboarding by following the runbook in `setup/interview.md` start to
finish. First check the `Onboarding status` in
`docs/developer/concept/concept-paper.md`:

- `not started` — run the interview from the start.
- `in progress` — resume from the first cluster not yet narrowed.
- `complete` — already onboarded. Do not silently re-run: ask the PO
  whether to revise specific parts of the concept paper or re-run from
  scratch, and proceed only on their choice.

`setup/interview.md` is the single source of truth — runbook, catalog,
closing steps. This skill is only the entry point.
