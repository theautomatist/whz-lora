# Feature

The project must be onboarded first: if the `Onboarding status` in
`docs/developer/concept/concept-paper.md` is not `complete`, run the
`/onboarding` skill before continuing — the feature process needs the
concept paper and the seeded feature registry.

Then run the feature process by following the runbook in
`setup/feature-process.md` start to finish, beginning at Step 1 (classify
the change as add, change or remove). `setup/feature-process.md` is the
single source of truth; this skill is only the entry point.
