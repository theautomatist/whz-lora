# Developer Documentation

Technical documentation for everyone who builds, reviews or extends this
project. The user-facing documentation is a separate site (`docs/user/`).

Structure:

- Onboarding — how this project works; start here
- Concept — the concept paper: the converged project definition
- Features — the feature registry: the single source of truth for what the product does
- Research — market analysis, sources, comparisons
- Architecture — system design, data model
- Diagrams — flows and timelines (Mermaid)
- Decisions — Architecture Decision Records (ADR)

This documentation grows with the code. Every notable architecture decision
is recorded as an ADR. See `CLAUDE.md`, pillar 2.
