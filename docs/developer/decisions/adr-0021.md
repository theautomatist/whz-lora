# ADR-0021 — Per-radiator parametric economic model with building archetypes

- Status: Proposed
- Date: 2026-06-12

## Context

The study must answer "is LoRaWAN heating control economical for building X/Y, and
what may it cost?" as a **dynamic** model. The design question (PO: *as few
parameters as possible, as many as necessary*) was which inputs to expose and how
to structure the verdict, given the preliminary research
([preliminary-research.md](../analysis/preliminary-research.md)).

## Decision

1. **Back-calculation.** The verdict is computed inversely: the maximum allowable
   system cost for payback ≤ horizon, compared to modelled cost → economical
   yes/no. This matches the PO framing ("rechne zurück von wirtschaftlich").
2. **Per-radiator framing.** Because annual benefit (∝ building heating energy)
   and system cost (∝ actuator count) both scale ~linearly with building size,
   size **largely cancels** in the yes/no. The verdict is a per-radiator question;
   building size is retained only for absolute € and for gateway sizing.
3. **Archetypes.** Building type is 3–4 named archetypes, each bundling a
   **thermal** profile (heating-energy intensity + attributable savings band) and
   an **RF** profile (attenuation class → gateway density). The two are correlated
   but set independently (a modern envelope is thermally efficient yet RF-hostile).
4. **Minimal operator inputs:** archetype, actuator count, energy price, horizon,
   hydraulic-balancing flag. Everything else (cost/valve, install, battery
   interval, discount rate, m²/radiator) is an archetype default or a constant,
   confirmed/pruned by a **sensitivity (tornado)** analysis.
5. **Headline metric** = simple payback (years) + the cost ceiling; NPV at a
   discount rate as a secondary check.

## Rationale

- **Parsimony (KISS).** The per-radiator cancellation collapses a dozen physical
  variables into ~3 operator knobs for the yes/no while staying faithful: a worked
  example (poorly-insulated building payback ~3 yr vs. modern ~16 yr) reproduces
  the literature's finding that smart control pays back in poorly-insulated, not
  well-insulated, buildings ([Schäuble et al. 2020](../analysis/preliminary-research.md)).
- **Archetypes** hide RF/thermal physics behind one choice — the operator never
  tunes attenuation in dB.
- The model is buildable on **published 868 MHz indoor path-loss parameters + link
  budget + the cost/benefit literature** — no fresh measurement campaign to start.

Alternatives rejected: (a) a full physical RF simulation (ns-3/FLoRa) as the
*primary* model — over-engineering for v1; simulators are reserved for **later
calibration** of the RF block only, against whz-lora measurements. (b) a bespoke
per-building model — not reusable; the archetype + scalars form generalises.

## Consequences

- Archetype default numbers (intensities, savings bands, gateway densities) are
  explicit assumptions in [model.md](../analysis/model.md), to be refined by the
  named deep-research targets (Phase 3) and calibrated against testbed RSSI/SNR
  (Phase 4).
- The per-radiator simplification is a **stated approximation**; where size matters
  (gateway step-changes, Buy-side hub ceilings) the model uses the absolute-€ path,
  not the per-radiator path.
- Several inputs are **PO policy, not facts** — perspective (whose money), horizon,
  energy price, discount rate — carried as overridable defaults (institutional
  owner-operator; 10 yr; €0.12/kWh; 3%).
