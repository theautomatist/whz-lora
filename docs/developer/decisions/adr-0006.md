# ADR-0006 — Onboarding as a Narrowing Funnel with a Concept Paper

- Status: Accepted
- Date: 2026-05-18

## Context

ADR-0003 introduced the first-run onboarding interview, which captured
answers into `setup/project-profile.md`. In practice the interview is
better understood as a convergence process: the project space starts
unbounded and must be narrowed down. A flat profile of answers does not
express this, and it duplicated the project's concept documentation.

## Decision

- The onboarding is framed as a funnel. The four interview clusters are
  four narrowing rounds; each ends with a cluster checkpoint where the main
  session synthesizes the bounded result and the PO confirms it.
- The single deliverable is a concept paper at
  `docs/developer/concept/concept-paper.md`, filled progressively as the
  funnel narrows. It replaces `setup/project-profile.md` and absorbs the
  former `concept/vision.md`.
- The concept paper carries an `Onboarding status` field. While it is not
  `complete`, the first-run onboarding runs or continues — this replaces
  the earlier file-existence signal.

## Rationale

- The funnel and the cluster checkpoints make the narrowing explicit and
  give the PO a confirmation point per cluster.
- One concept paper instead of a separate profile and vision file removes
  duplication (KISS) and produces a genuine, readable project document.
- A status field is a more reliable signal than file existence — it
  correctly handles an interrupted, partially filled paper.

## Consequences

- The onboarding output is a single, audience-facing concept paper that
  lives in the developer documentation.
- `setup/project-profile.md` and `concept/vision.md` no longer exist.
- ADR-0003 still holds (the first-run interview); this ADR refines its
  output model.
