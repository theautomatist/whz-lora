# ADR-0012 — Verification Profile per Project

- Status: Accepted
- Date: 2026-05-19

## Context

ADR-0004 made a "deployment smoke test" — the built software deploys and
starts cleanly in a container — a fixed step of the Verify stage and of the
Definition of Done. That assumes a deployable service. A library does not
"deploy and start"; a CLI tool is installed and invoked; firmware is built
and flashed to hardware. The template aims to serve these project types
too, so a single hard-coded verification step does not fit.

## Decision

- The fixed "deployment smoke test" is replaced by a per-project
  **verification check** — an artifact-appropriate confirmation that a
  build is sound, decided during onboarding and recorded in the concept
  paper:
  - service / web app — deploy in a container, the main paths respond;
  - CLI tool — install in a clean environment, run it;
  - library — build, install as a dependency, import and call it;
  - firmware — build the image; where a hardware rig exists, flash and
    confirm it reports.
- `CLAUDE.md` (Verify step, Definition of Done, Releases, Testing)
  references "the project's verification check" instead of a deployment
  smoke test.
- Container isolation for unit, integration and end-to-end tests is
  unchanged — that is universal.

## Rationale

- One parameterised concept replaces one wrong assumption — KISS-neutral
  and more correct, no extra moving parts.
- The onboarding interview (theme 15) already asks what confirms "it runs";
  this decision gives that answer a defined home and a defined use.
- The process becomes artifact-agnostic. A hardware project still needs
  project-specific test infrastructure (a hardware-in-the-loop rig); that
  is an onboarding and infrastructure concern, not something a generic
  template can ship.

## Consequences

- The lifecycle fits services, CLI tools, libraries and firmware without
  bending the process per project.
- The verification check must be defined at onboarding; an undefined check
  is an onboarding gap.
- ADR-0004 still holds (testing pyramid, release verification); this ADR
  generalises its deployment smoke test.
