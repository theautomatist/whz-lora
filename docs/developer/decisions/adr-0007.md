# ADR-0007 — Onboarding Skill and Setup-Recommendation Step

- Status: Accepted
- Date: 2026-05-18

## Context

The onboarding ran only as a First-Run trigger embedded in `CLAUDE.md` — it
could not be invoked deliberately or re-run, and was not discoverable.
Separately, the onboarding tuned agents and gates but never proactively
told the PO which MCP servers, skills or tools would help this specific
project.

## Decision

- The onboarding gains an explicit entry point: an `onboarding` skill,
  invoked by `/onboarding`. The `CLAUDE.md` First-Run trigger remains as the
  automatic safety net and now simply invokes the skill. `setup/interview.md`
  stays the single source of truth — runbook plus catalog.
- The skill also handles deliberate re-onboarding: if the concept paper's
  `Onboarding status` is already `complete`, it asks the PO whether to
  revise parts of the paper or re-run from scratch.
- A new interview theme — 21, Setup Recommendations — runs after the stack
  is decided. The `research` agent finds, tailored to the stack and the
  project profile, the MCP servers, skills and tools that would measurably
  help the project. The PO accepts or declines each; accepted items are
  wired in during the closing step. Findings are recorded in
  `setup/stack-knowledge.md`.

## Rationale

- Two complementary entry points: the skill gives explicit, discoverable,
  re-runnable access; the First-Run trigger keeps onboarding automatic for
  a user with zero prior knowledge. Neither alone is sufficient.
- A thin skill over the existing runbook avoids duplicating the interview
  logic (KISS).
- Setup recommendations turn the onboarding from purely "ask questions"
  into "also actively improve the project's tooling". Recording them in
  `stack-knowledge.md` compounds the knowledge across projects.
- Setup research is consolidated into one theme rather than scattered as
  pervasive live research, to avoid constant interruption (KISS).

## Consequences

- `/onboarding` is available anytime; the project can be re-onboarded
  deliberately.
- The onboarding produces, besides the concept paper, a curated set of
  setup enhancements.
- The interview has 22 themes; themes 20 and 21 are research-driven.
